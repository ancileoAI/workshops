## Why Do We Test?

### The Cost
Bugs are expensive — and later we catch them, the more they cost:

- **In development**: ~$100 — just your time and a quick fix.
- **In QA/testing**: ~$1,500 — now someone else is involved, maybe bugs need retesting etc.
- **In production**: $10,000+ — real users are affected, revenue may be lost, and reputation takes a hit.

**Benefits of Testing**
1. **Early Bug Detection** - Catch issues before they reach users
2. **Living Documentation** - Tests show how code should work
3. **Refactoring Safety** - Change code with confidence
4. **Better Design** - Writing testable code leads to better architecture
5. **Team Collaboration** - Tests communicate intent to other developers

### The Mindset

```
"Testing is not about finding bugs, it's about building confidence"
"It isn’t just a task — it’s a way of thinking about your code and how it serves the people who use it."
```

Key principles:
- Test behavior, not implementation
- Start with the simplest case
- Think about edge cases
- Write tests as if you're the user of your code
